# UX Principles — AI エントリーポイント

このリポジトリはClaude CodeなどのAIエージェントがUX原則に基づいたチェックと改善提案を行うためのMCPサーバーです。

---

## クイックリファレンス — MCPツール

| ツール | 用途 | 例 |
|--------|------|-----|
| `get_principle("nielsen")` | ニールセンの10原則を取得 | UIレビュー前の原則確認 |
| `get_principle("gestalt")` | ゲシュタルト原則を取得 | レイアウト設計時の確認 |
| `get_principle("all")` | 全原則を取得 | 包括的なUX学習 |
| `check_ux("html", <content>)` | HTMLのUXチェック | 生成UIの一次スクリーニング |
| `check_ux("component", <content>)` | コンポーネント単位のチェック | ボタン・フォームの評価 |
| `check_ux("flow", <content>)` | ユーザーフローのチェック | 画面遷移の評価 |
| `check_ux("description", <content>)` | 画面説明文のチェック | 要件定義・設計書の評価 |
| `suggest_fix(["n01", "g05"])` | 違反への改善提案 | check_ux後の修正対応 |

## UX原則サマリー

### ニールセンの10原則（重要度順）

| ID | 原則 | 重要度 | 一言サマリー |
|----|------|--------|------------|
| n01 | システム状態の可視性 | 🔴高 | 今何が起きているか常に見せる |
| n03 | ユーザーコントロールと自由 | 🔴高 | 間違えても簡単に戻れる |
| n04 | 一貫性と標準 | 🔴高 | 同じ操作・同じUI |
| n05 | エラーの防止 | 🔴高 | エラーが起きないように設計 |
| n09 | エラーの認識・診断・回復 | 🔴高 | エラーは明確・具体的・建設的に |
| n02 | 実世界との一致 | 🟡中 | ユーザーの言葉を使う |
| n06 | 記憶の最小化 | 🟡中 | 覚えさせない |
| n08 | 美的で最小限のデザイン | 🟡中 | 不要な情報を排除 |
| n07 | 柔軟性と効率性 | 🟢低 | 上級者にもショートカットを |
| n10 | ヘルプとドキュメント | 🟢低 | 必要な時にすぐ見つかる |

### ゲシュタルト原則（視覚認知）

| ID | 原則 | 重要度 | 一言サマリー |
|----|------|--------|------------|
| g01 | 近接 | 🔴高 | 近いものは関連している |
| g02 | 類似 | 🔴高 | 似ているものは同じグループ |
| g05 | 図と地 | 🔴高 | コントラストで前景と背景を分ける |
| g04 | 連続 | 🟡中 | 視線の流れに沿ってレイアウト |
| g07 | プレグナンツ | 🟡中 | シンプルな形・グリッドに揃える |
| g03 | 閉合 | 🟢低 | 不完全でも形として認識できる |
| g06 | 共通運命 | 🟢低 | 同じ動きで関連性を示す |

## タスクベース読み込みガイド

| タスク | 参照すべきファイル |
|--------|-----------------|
| HTMLレビュー | `check_ux("html", ...)` → 違反があれば `suggest_fix(...)` |
| 画面設計の議論 | `get_principle("nielsen")` + `get_principle("gestalt")` |
| フォーム設計 | n05（エラー防止）・n06（記憶最小化）・g01（近接）を重点確認 |
| 一覧・テーブル設計 | n07（効率性）・n08（最小限）・g04（連続）を重点確認 |
| エラー画面設計 | n09（エラー回復）・n01（状態可視性）を重点確認 |
| ナビゲーション設計 | n04（一貫性）・n06（記憶最小化）・g02（類似）を重点確認 |

## melta UIとの組み合わせ方

このリポジトリは**UX（使いやすさ）**を担保します。
melta UIは**UI（見た目の一貫性）**を担保します。

```
check_ux("html", <content>)      # UX観点のチェック（このリポジトリ）
check_rule("shadow-lg text-black") # UI観点のチェック（melta UI）
```

両方を通過したUIが「見た目も使い勝手も良い」状態です。

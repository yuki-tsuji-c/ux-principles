# HTMLチェック観点

HTMLファイルをUX観点でチェックする際の確認ポイント。

## 優先チェック項目

### フォーム
- [ ] 必須フィールドに`required`属性があるか
- [ ] `placeholder`で入力例が示されているか
- [ ] エラーメッセージの表示領域が用意されているか
- [ ] submitボタンは処理中にdisabledになるか

### ボタン・インタラクション
- [ ] ローディング状態が表示されるか
- [ ] 操作後のフィードバック（成功/失敗）があるか
- [ ] 破壊的操作（削除）に確認があるか

### ナビゲーション
- [ ] 現在地（パンくず・アクティブメニュー）が分かるか
- [ ] 戻るボタン・キャンセルが存在するか

### アクセシビリティ（WCAG AA）
- [ ] `alt`属性が画像に設定されているか（装飾画像は`alt=""`）
- [ ] `aria-label`がアイコンボタンに設定されているか
- [ ] フォーカス順序が自然か
- [ ] テキストと背景のコントラスト比が4.5:1以上か
- [ ] `outline: none`でフォーカスリングを消していないか（代替スタイルがあればOK）
- [ ] `<input>`に`<label>`が関連付けられているか（placeholderだけはNG）
- [ ] `<div onclick>`ではなく`<button>`を使っているか
- [ ] 見出しレベルが飛んでいないか（h1→h3はNG）
- [ ] `<nav>`, `<main>`, `<header>`, `<footer>`などのランドマークを使っているか
- [ ] エラーが色だけでなくテキストでも伝わるか

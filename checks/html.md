# HTMLチェック観点

HTMLファイルをUX観点でチェックする際の確認ポイント。

## 優先チェック項目

### フォーム
- [ ] 必須フィールドに`required`属性があるか
- [ ] `placeholder`で入力例が示されているか
- [ ] エラーメッセージの表示領域が用意されているか
- [ ] submitボタンは処理中にdisabledになるか

### ボタン・インタラクション
- [ ] ローディング状態が表示されるか
- [ ] 操作後のフィードバック（成功/失敗）があるか
- [ ] 破壊的操作（削除）に確認があるか

### ナビゲーション
- [ ] 現在地（パンくず・アクティブメニュー）が分かるか
- [ ] 戻るボタン・キャンセルが存在するか

### アクセシビリティ
- [ ] `alt`属性が画像に設定されているか
- [ ] `aria-label`がアイコンボタンに設定されているか
- [ ] フォーカス順序が自然か
